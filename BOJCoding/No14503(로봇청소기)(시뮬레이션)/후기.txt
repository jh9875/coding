문제중에서 이런 문구가 있었다.
네 방향 모두 청소가 이미 되어있거나 벽인 경우에는,
바라보는 방향을 유지한 채로 한 칸 후진을 하고 2번으로 돌아간다.

여기서 나는 제대로 못보고
그냥 함수를 return하면 되는줄 알았는데 방향을 유지한체로 다시한번
함수를 호출해 줬어야 한다.


그리고 4가지 방향을 모두 검사하는 코드를 보면
for(int i =0; i <4; i++)	//4번 턴
{
	d =(d+3) %4;			//왼쪽으로 턴.
	if(0 <=r+dirR[d] && r+dirR[d] <N
			&& 0 <=c+dirC[d] && c+dirC[d] <M
			&& map[r+dirR[d]][c+dirC[d]] ==0)
	{
		search(r+dirR[d], c+dirC[d], d);
		return;
	}
			
}

//back

이러한데
search함수를 호출하고 나면 return을 해줘야 4방향 중 한곳이상이 갈수있는 곳임을
확인할 수 있다.
만약 저기서 반환되지 않으면 
밑에있는 뒤로 이동하는 함수를 호출하게 된다.
